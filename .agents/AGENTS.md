# Reglas del Proyecto (Workspace Rules)

## Flujo de Git y Ramas

- **PROHIBIDO hacer commits y pushes directos a la rama `main`.**
- Los únicos commits y pushes permitidos en `main` son aquellos generados automáticamente o de forma manual para **Releases oficiales** del proyecto.
- Todo desarrollo, refactorización o documentación debe realizarse en ramas de features (`feat/*`), bugs (`fix/*`) o de soporte, y ser integrados preferentemente a la rama `develop` mediante Pull Requests.
